# Adding Contract Files in the Web3 Tab

## Prerequisites:

- A Public/Private GitHub repository

### Step 1: Access the Web3 Tab

1. Navigate to the application interface.
2. Click on the **Web3** tab located in the main navigation menu.

### Step 2: Add Contracts

1. Locate the **Add Contracts** button on the Web3 tab interface.
2. Click on the **Add Contracts** button.

### Step 3: Upload Contract Files

1. In the pop-up or section that appears, locate the upload option labeled **Upload .sol Files or .zip**.
2. Use the provided interface to browse your local files.
3. Select the desired `.sol` (Solidity) file(s) or a `.zip` file containing multiple contract files.
4. Confirm the upload by clicking the appropriate button, usually labeled **Upload** or similar.

### Step 4: Finalize Adding Contract Files

1. After uploading, click the **Add Contract File** button to finalize the process.
2. Ensure the uploaded files appear in the contract list or designated area within the Web3 tab.

## Notes

- Ensure that your `.sol` or `.zip` files are properly formatted and contain valid contract definitions.
- Double-check the uploaded files to confirm they meet the applicationâ€™s requirements.
- If you encounter errors during the upload process, review the error message and verify your files accordingly.

## Troubleshooting

### Issue: "File format not supported."

**Solution:** Ensure you are uploading `.sol` or `.zip` files only.

### Issue: "Upload failed."

**Solution:** Check your internet connection and retry.

### Issue: "Contract not appearing after adding."

**Solution:** Verify if the correct file was uploaded and added by revisiting the **Add Contracts** section.
